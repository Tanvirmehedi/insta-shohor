Each browser has its own JavaScript engine. Browsers read JavaScript languages in their respective JavaScript engines. exp -v8, chakra, spider monkey etc. Converts to binary code through compiler and makes the machine readable. An interpreter in the browser reads over the JavaScript code, interprets each line, and runs it.

JavaScript has runtime model eventloop. Responsible for executing substacks, collection and processing events and code. The single threat language makes it work like a multi-threat. It makes sure that the call stack is not emptied. If it becomes empty then add a new function from the event queue. And if not then the current function is processed.
